

# Пошук товарів по назві + вибір варіанту

## Що зробимо

Замінимо ручне введення Product ID / Variant ID на зручний пошук по назві товару з Shopify. Після вибору товару -- покажемо список його варіантів для вибору.

## Зміни

### 1. Діалог товарів -- новий UX

Замість двох інпутів (Product ID, Variant ID) зробимо:

1. **Поле пошуку** -- вводиш назву товару, підвантажуються результати з Shopify (через існуючу edge function `shopify-products`)
2. **Випадаючий список результатів** -- показує назву товару, SKU першого варіанту, зображення
3. **Після вибору товару** -- якщо у товара >1 варіант, показуємо список варіантів (назва опцій, SKU, ціна) з можливістю обрати конкретний або "Всі варіанти"
4. **Кнопка "Додати"** -- зберігає `shopify_product_id` + опціонально `shopify_variant_id`

### 2. Кешування списку товарів

Завантажимо всі товари з Shopify один раз при відкритті діалогу товарів (edge function вже підтримує пагінацію і повертає всі товари). Фільтрація по назві буде клієнтська -- швидко і без зайвих запитів.

### 3. Відображення доданих товарів

В списку доданих товарів замість сухих ID покажемо **назву товару** та **назву варіанту** (з кешованих даних Shopify).

## Технічні деталі

- Використовуємо `supabase.functions.invoke("shopify-products")` для завантаження товарів
- Фільтрація клієнтська: `product.title.toLowerCase().includes(query)`
- Після вибору товару показуємо `product.variants` масив
- Зберігаємо `String(product.id)` та `String(variant.id)` в БД як і раніше
- Без змін в БД чи edge functions
