

# Редизайн картки "Помилки валідації" на дашборді

## Проблема

Зараз картка показує **загальну кількість помилок за весь час** (10065) — це накопичувальне число, яке не дає корисної інформації. Кожна генерація фіду додає нові записи в `validation_errors`, і число тільки росте.

## Що зробимо

Замінимо безглузду картку на корисну інформацію — покажемо помилки **тільки з останньої генерації** для кожного маркетплейсу. Це дасть зрозуміти, скільки товарів мають проблеми **прямо зараз**.

### Зміни в `src/pages/Dashboard.tsx`:

1. **Перейменувати** "Помилки валідації" → "Проблемні товари (остання генерація)"
2. **Змінити запит** — замість `count(*)` всієї таблиці, рахувати помилки лише для останнього `feed_log_id` кожного маркетплейсу (тобто джойн з `feed_logs` по останньому запису)
3. **Показати розбивку по маркетплейсах** — невелика таблиця або список: Rozetka — X помилок, Maudau — Y, Epicentr — Z
4. **Прибрати** велику безглузду цифру "10065"

### Логіка запиту:

- Беремо останній `feed_log` для кожного маркетплейсу (де `status = 'success'`)
- Рахуємо `validation_errors` з відповідними `feed_log_id`
- Показуємо результат по кожному маркетплейсу окремо
