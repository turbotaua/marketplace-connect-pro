

# Recurring промо + годинне планування + баг-фікси

## 1. Зміни в базі даних

Додати нові колонки до таблиці `promotions`:

- `is_recurring boolean default false` -- чи повторюється акція
- `recurrence_pattern text` -- 'daily' або 'weekly'
- `recurrence_day_of_week int` -- день тижня (0=Нд, 1=Пн...6=Сб), для weekly
- `start_time text` -- час початку (HH:MM), наприклад "10:00"
- `end_time text` -- час завершення (HH:MM), наприклад "18:00"

Для одноразових акцій `starts_at`/`ends_at` зберігаються як є, але тепер з підтримкою годин (не тільки дат).

Для recurring: `starts_at` = дата першого запуску, `ends_at` = дата коли recurring закінчується взагалі (опціонально, може бути null = безстроково). `start_time`/`end_time` = щоденний/щотижневий вікон часу.

## 2. UI форми створення промо (`CreatePromoDialog`)

- Додати Switch "Повторювана акція"
- Якщо recurring ON:
  - Select "Щоденно" / "Щотижнево"
  - Якщо щотижнево -- Select дня тижня (Пн-Нд)
  - Два інпути часу: "Початок" (HH:MM) та "Кінець" (HH:MM) -- використати `<Input type="time">`
  - Calendar "Діє з" (з якої дати починається recurring)
  - Calendar "Діє до" (опціонально, до якої дати)
- Якщо recurring OFF (як зараз):
  - Calendar з додаванням часу -- `<Input type="time">` поруч з Calendar для точного часу

## 3. Логіка `isActive` в таблиці

Оновити функцію `isActive()`:
- Для одноразових: як зараз, але з урахуванням часу
- Для recurring: перевіряємо чи зараз потрапляє в вікно (правильний день тижня + час між start_time і end_time + в межах starts_at..ends_at)

## 4. Відображення в таблиці

- В колонці "Період": для recurring показати "Щотижнево, Пн 10:00-18:00" замість дат
- В колонці "Статус": додати бейдж "Повторювана"

## 5. Баг-фікс: скрол в пошуку товарів

`ScrollArea className="max-h-56"` не працює коректно. `ScrollArea` потребує явної фіксованої висоти. Заміна на `h-56 overflow-y-auto` або обгортка з фіксованою висотою.

## 6. UI/UX фікс: додані товари

З другого скріншоту видно що довга назва варіанту обрізається погано і кнопка видалення занадто маленька. Покращимо:
- Кнопка видалення більша і помітніша
- Текст назви товару та варіанту на окремих рядках
- Зображення товару в списку доданих (як в пошуку)

## 7. Інші QA фікси

- Статус-бейджі можуть показуватися одночасно (expired + вимкнена) -- додати `else` логіку
- Валідація: `end_time` > `start_time` перед збереженням
- Порожній стан промо-таблиці: перевірити що `promotions?.length === 0` працює коректно (може бути `undefined`)
